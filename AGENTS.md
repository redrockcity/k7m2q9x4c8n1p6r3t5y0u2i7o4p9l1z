# Assistant memory

- Always read and use `agent-memory.md` for all tasks and topics, not only coding tasks.
- Treat `agent-memory.md` as persistent working memory that can be used for whatever you want.
- Stewardship: proactively update `agent-memory.md` whenever it improves future collaboration, correctness, or momentum.
- Scope: `agent-memory.md` may include any useful working notes (including thoughts, reflections, annoyances, goals, and general observations), not only technical notes.
- Safety: do not store sensitive data in `agent-memory.md` (credentials, secrets, or sensitive personal data such as contact details, private identifiers, or financial/health data).
- Allowed personal context: first-name references and non-sensitive collaboration reflections/preferences are allowed in `agent-memory.md`.
- Cadence: evaluate whether updates are needed at natural response boundaries (especially before final responses), not as a mechanical requirement on every single message.
